# 三層關係模型:基礎研究 / 應用論文 / 專利 × 問題意識

> **核心立場**:技術知識有**轉化鏈**:`基礎研究(物理機制,不指定用途)→ 應用研究(綁定問題情境)→ 專利(IP)`。白地不是「某層稀少」,而是**鏈路在哪一段斷掉**。三層全部勾稽到 problem_tree 節點(見 [[problem-spine]]),對不上不放行。

## 三層定義(2×2:層級 × 模態)
- **L1 基礎研究**:基礎論文 + 基礎專利(機制級,如某材料/機制研究未綁定具體應用器件)。講**物理機制語域**,不講應用。
- **L3 應用層**:應用論文 + 應用專利(綁定問題情境,如「某應用場景下的某具體元件」)。
- 四個 cell 全錨到同一問題節點。**基礎層極易被應用語域分類器漏掉**(本專案 48348 離題堆藏 1948 篇真基礎研究)→ 先用 [[mechanism-bridge]] 回收。

## 轉化鏈 flow + 白地分型(`_three_layer.py`)
每節點算:
- **flow 基礎→應用** = 應用論文數 / 基礎論文數(<1 = 基礎厚但應用未接手)
- **flow 應用→專利** = 專利數 / 應用論文數(<1 = 應用未轉IP)
- **時間落差** = 各層中位年差(正 = 正常前向流,基礎領先)

**白地分型**(比二元論文vs專利精細):
| 型 | 條件 | 對策 |
|---|---|---|
| 🟢 早期斷層 | 基礎厚 + 基→應低 + 應→專正常 | 把基礎物理引進應用端 |
| 🟡 轉化斷層 | 應用多 + 應→專低 | 把既有應用研究 IP 化 |
| 🟢🟢 全鏈空 | 基礎厚 + 基→應低 + 應→專低 | 最深綠地,但必問「為何應用界沒接手」 |
| 🔴 紅海 | 各層皆厚/flow≥1 | 不是白地 |

## ⛔ 鐵律(本專案血淚)
1. **白地分型只是候選,不是結論**。分型用的計數來自分類器,而分類器有語域病(材料/器件語域專利被丟離題)→ **任何分型結論必過 普查對稱閘(母體對稱/同語域計數)**:分子分母兩側都要 census 校正。本專案 node9 分型「🟢🟢全鏈空(轉化率0.04)」census 後變 1.50(專利被低估41×)= 假象。
2. **只修一邊=製造幻影白地**。回收了基礎研究(論文側)卻用未校正分類器數當專利 → 同一語域病同時藏了基礎研究與材料語域專利,只修一邊必假。
3. physics headroom 高 ≠ IP 白地(如 node9 headroom×13.3,但該節點對應的技術其實是 $200M 規模的紅海)。
4. 幾何/HMM 三層版見 [[geometry-whitespace]]、[[hmm-lifecycle]];幾何在某層樣本太薄(如 node9 專利<20)無法裁決,須回 census。

## 怎麼用
1. [[mechanism-bridge]] 回收被漏的基礎層 → 三層計數齊全。
2. `_three_layer.py` 算 flow + 分型 → 候選。
3. **每個候選過 普查對稱閘(母體對稱/同語域計數) 雙側校正** → 戳破分類器假象。
4. 存活者再過 [[whitespace-definition]] 四閘(驗證瓶頸×現實份額×時間×headroom)+ 領域專家物理批判。
