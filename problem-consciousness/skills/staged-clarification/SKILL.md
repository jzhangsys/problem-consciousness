# 軟問題階段式澄清(模糊 → 可操作的決策判準)

## 立場
不是每個問題都是可量化瓶頸。規範/概念/實務類問題(個人、事業、策略、價值、動機)**無法、也不該**被硬套「指標+單位+尺度」——但它們**同樣不准停在模糊**。對這類問題,「清楚」= **可操作化的決策判準**,不是物理單位。模糊不因為它是軟問題就被赦免;它被**階段式**逼清楚。
★鐵律:軟問題**不打回**(不當「模糊主題」剔除),而是進 L0–L8 逐階澄清。硬給它假數字(偽精確)同樣不准。

## 何時走這條路(由 [[meta-problem-layer]] 認識論狀態路由)
- **概念**(「成功/清楚/我要的」指什麼)、**規範**(該不該、優先什麼)、**實務**(該做什麼、改哪個決策)→ 走本路徑。
- **事實/因果-可量化**(有物理/數值瓶頸)→ 走 [[problem-identification]](指標+單位+尺度)。
- 一個問題常同時含兩類:量化部分交 problem-identification,規範/概念/實務部分交本路徑,最後合流。

## L0–L8(軟問題版;每階有產物才放行,禁跳層)
| 階 | 軟問題核心問 | 產物 | 取代了量化版的 |
|---|---|---|---|
| L0 現象 | 說出模糊的卡點/渴望/不適(不含解法) | phenomenon(如「事業模糊」) | — |
| L1 邊界 | 對誰、哪個決策場景、什麼範圍 | context | — |
| L2 構念 | 「清楚/成功/我要」在這裡具體指什麼 | 操作化定義 + 反概念 | 代理指標 |
| L3 判準 | 怎樣**算**「清楚了」——可觀察的決策判準 | decision criteria | **量測 → 決策判準** |
| L4 根因 | 為何模糊(症狀 vs 因) | 根因分類(見下) | 機制/因果鏈 |
| L5 方向 | 有哪些介入(不排名、不承諾最佳) | path(每條對應根因) | — |
| L6 驗證 | 怎麼知道有效:決策變快/一致/後悔變少 | soft 驗證 + 反證 | 實驗/證偽設計 |
| L7 落地 | 進真實生活/組織並穩住 | implementation | — |
| L8 停止 | 何時算「夠清楚」,停止再糾結 | stopping rule | — |

## 三種「模糊根因」(L4 必分——分錯就空轉)
- **缺資訊**:不知道選項/後果 → 補查、補證。
- **缺取捨**:知道選項,但價值互相衝突、優先序未定 → 逼出取捨與排序。**這常是真根因**(貼近「不清楚自己要什麼」)。
- **缺承諾**:其實已經清楚,卡在不敢定 → 需 L8 停止規則 + 做決定,而非再分析。
★把「缺取捨/缺承諾」誤當「缺資訊」去狂查資料,是軟問題最常見的空轉;分對根因才選對介入。

## 「可操作化」判準(取代量化門檻)
一個軟問題澄清到位 = 它產出一個**可判定的決策判準**:給定選項,能說出「選 A 還是 B、依據什麼、事後怎樣算對」。若仍是「看情況/都可以/再說」→ **尚未澄清**,退回 L2/L3,而不是放行。

## 紀律
- 沿用 [[problem-level-processing]] 的 C0–C8 結論等級與防跳層:軟問題同樣不准把 L3 判準當 L4 根因、不准把渴望當結論、結論等級不得超過證據等級。
- 誠實面對不可量化:用決策判準,不用假數字冒充嚴謹。
- 涉重大人生/事業/價值決策 → L8 掛人類 gate(承 problem-level-processing R6)。
- 承 [[meta-problem-layer]] 路由;與 [[problem-identification]] 並列;統一元紀律見 [[problem-level-processing]]。
