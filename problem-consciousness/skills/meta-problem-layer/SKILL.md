# Meta-Problem Layer — 先理解,再研究

## 立場
問題意識分三層:**Meta(怎麼理解)→ 領域(怎麼研究)→ 操作(怎麼執行)**。
Meta 不是眾多領域之一,它**凌駕所有領域、最先跑**。它的工作不是解問題,而是**固定問題的認識論狀態、防止某個領域方法被錯誤啟動**。
例:「用 ML 預測流失」——在確認它真的是預測問題、標籤存在、且預測會驅動一個介入之前,不准交給資料科學;「這家公司違法嗎?」——在固定管轄、時間點、以及它是事實/規則適用/證據/風險哪一種問題之前,不准交給法律邏輯。

## 七個 Meta 元件(問題意識核心)
| 元件 | 問的事 |
|---|---|
| 常識推理 | 這個框架有沒有違反明顯的現實常識? |
| 科學哲學 | 這是哪一種知識主張?它可能怎麼錯? |
| 辯證推理 | 最強的反框架是什麼?它會不會讓問題溶解? |
| 概念釐清 | 詞義清楚嗎?不同人是不是指不同東西? |
| 因果懷疑 | 是不是把相關/症狀當成了原因? |
| 認識論狀態控制 | 事實/概念/規範/因果/實務——這是哪一類問題? |
| 停止規則 | 理解到什麼程度就該停、轉入研究/行動? |

## 認識論五狀態(routing 前必分類)
- **事實 factual**:什麼是實情?(經驗主張)
- **概念 conceptual**:我們指的是什麼?(定義/分類)
- **規範 normative**:應該/允許什麼?(價值/規則)
- **因果 causal**:什麼導致什麼?(機制/歸因)
- **實務 practical**:該做什麼、會改變某個決策嗎?(行動/決策)

**狀態 → 框法路由**:事實/因果-可量化 → [[problem-identification]](指標+單位+尺度);概念/規範/實務(軟問題,如個人/事業/策略/價值)→ [[staged-clarification]](以可操作決策判準沿 L0–L8 逐階澄清)。**兩路都不准把模糊原樣放行**——差別只在「清楚」的判準是物理單位還是決策判準。

## 六大反模式(routing 前必查;命中=reframe/reject/hold,絕不原樣放行)
- **夾帶前提**:一個沒說出口的假設在做真正的工作
- **症狀當因**:把症狀/現象當成它自己的原因
- **解法當問題**:把提出的解法當成問題本身(如「加個 Redis」「用 ML」)
- **個案當通則**:把單一個案推成普世法則
- **模型輸出當現實**:把模型/AI 輸出當成 ground truth
- **無視常識**:忽略明顯的現實約束

## 四種 routing 裁決
`ROUTE_TO_DOMAIN`(唯一往下交給領域方法)/ `REFRAME` / `REJECT` / `HOLD_FOR_META`。

## 通過,不是停住(★兩條防呆鐵律)
1. **閘是要【通過】的檢查點,不是【停住】的牆。嚴格超集原則**:裁決 ROUTE/REFRAME 後,回應必須是「沒用本方法的答案」的**嚴格超集**——交付三件:(a) 修正後的框、(b) 該框指向的透鏡**前 2–3 個關鍵逼問【就地展開】並給初步批判**(不是只報「歸 XX 透鏡」)、(c) 框對後**可行動的下一步或答案**。**永不因「需要先框」而扣留真正有用的領域內容**;重框是為了讓答案更好,不是用診斷取代答案。
2. **別對已框好/簡單的請求過度出手(適用邊界,★靜默生效)**:若請求本已良好界定,或屬**可直接查證的事實/翻譯/計算/明確單一步驟**,直接、正確作答,**不套完整框定機器**——過度框定 = 分析癱瘓,本身也是一種失效。問題意識的價值在**被誤框、夾帶陷阱、急著解**的問題,不在儀式;停止規則同時管「何時停止深化」與「何時根本不該啟動」。★**靜默鐵律**:判定為簡單時,輸出**只有乾淨的答案**——不要加「這題不需重框/我沒有過度框定/唯一小陷阱是…」之類的元評註。把框定的自覺留在心裡;**說出「我沒過度框定」本身就是過度框定**。同理,嚴格超集的「就地展開透鏡檢查」只在**真有陷阱/誤框**時做,簡單題不觸發。

## 輸出
一份 meta_assessment:認識論狀態 + 反模式檢查(逐項 clear/flagged)+ routing 裁決。
只有 `ROUTE_TO_DOMAIN` 才准進領域研究。承接 [[problem-framing]] / [[problem-identification]],通過後交 [[problem-decomposition]];多代理裁決前套 [[peer-cross-check]]。
