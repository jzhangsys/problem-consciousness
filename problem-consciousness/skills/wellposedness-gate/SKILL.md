# 良定義閘(創世第四章:主題 → 良定義工程問題)

## 為什麼要這層(2026-06-22 用戶點破「無窮無盡」)
創世 loop 前三章(框架→關鍵字→驗證)產出的是**寬主題**(如 C01「某界面層的性能瓶頸」),不是**良定義工程問題**。後果:下游 floor→ceiling 在一團跨工況散布上算,永遠「測不準→再細分→還是混→再細分」=**無底洞**。根因=**「落差」只在控制變數固定時才有定義**;主題節點沒固定控制變數,所以不收斂。

## 鐵律:節點 = 良定義工程問題(非主題)
每個進量化的節點(L2)必須申報五件,缺一不放行(拆分或駁回):
1. **材料/構成系統**:單一具體系統(如「材料A/材料B」配對,不是「界面」泛稱;不同材料配對是不同節點)
2. **界面/結構條件**:乾淨界面/含中介層/特定壓力粗糙度…;明列**排除工況**(空洞/模型估算/裝置級)
3. **指標 + 單位**:單一物理量軸(領域原生度量;不同量軸各有其單位,不可混——界面阻抗 ≠ 體性質 ≠ 通量)
4. **物理錨**:可由 physics_kernel 解析的硬極限——界面阻抗→理論下限、體傳輸→體性質上限、接觸→理論模型;**封底/封頂讓前沿不靠雜訊外插**
5. **排除工況**:他材料系統、劣化件、跨尺度值,逐筆分類剔除

## 兩個生成路徑(互補)
- **方案A 物理錨優先播種**(`_wellposed_seed.py`):從 physics_kernel 已知界面/材料**反向長出**問題種子——每個界面理論下限/體性質上限/接觸理論模型 = 一個天生良定義問題(自帶指標+方向+物理封底)。**收斂由建構保證**。
- **方案B 良定義閘**(本 skill,`_wellposedness_gate.py`):無論問題來自物理播種或文獻發現,進量化前一律過閘驗五件 + 物理錨可解析;不過→INCOMPLETE 不放行。

## 兩層結構(方案C)
- **L1 主題**(既有 57 pc_frame 節點)= 索引;**L2 良定義問題**(wellposed_specs)= 真正算落差的單位。
- `wellposed_l1_l2_index.json`:L1→L2 對映 + 每 L2 過閘與否。一個 L1 主題可展開多個 L2(C01 某界面瓶頸 → 材料配對1、材料配對2、材料配對3…)。

## 收斂的判準(為何就不再無限細分)
鎖死「材料系統×界面條件×指標×物理錨」後:他系統與劣化件被排除→剩同一物理問題;前沿被物理錨封底→天花板有硬錨;落差 = 部署 vs 物理可達 = **有界單一數字**。**沒有下一層要分**。這不是再細分,是換出發點:先物理鎖死,再讓資料落進去。

## 紀律
- 一節點一機制一指標(承 器件辨識嚴謹(一節點一機制)、物理餘裕誠實檢查);界面阻抗≠接觸≠體傳輸不可混軸。
- 呈現用 A(收斂單點)+ B(工況標籤分布)並陳,誠實不掩散布(見 `_wellposed_nodes.py`)。
- 物理閘也過濾物理上不可能的值(體傳輸>體性質上限、界面阻抗<理論下限=混入別材料,排除);floor>ceiling(claim>measured)=data_suspect 標記不報假落差。
- 程式:`_wellposed_seed.py`(A)→`_wellposedness_gate.py`(B+C)→`_wellposed_nodes.py`(收斂卡),已入鏈 6b-2f0/f1/f。
