# [C·批判] 套套邏輯批判

## 立場
假設驗證是「領域過濾偽裝成問題意識」。實證:23% 的有效組合,其 on-topic 標題只是**回聲 query 詞**(查某瓶頸詞→標題出現同一詞,廢話)。就算其餘有新詞,ON 正則只要任一泛領域詞就過——這測的是「是不是這個領域」,不是「有沒有處理這個瓶頸」。這跟之前離題(如牙科)專利同一病根,只是往後移一步。

## 攻擊點
1. **回聲檢測**:對每條 valid,計算 on-title 是否只含 query 詞、無新增資訊。回聲比例高=驗證無鑑別力,駁回。
2. **領域 vs 瓶頸**:抽查 valid 的 abstract——它真的處理該 problem_id 的瓶頸機制,還是只是同領域泛文?只是同領域=FAIL。
3. **量化缺席**:valid 的 abstract 有沒有該瓶頸的量化數字?沒有=領域過濾(接領域量化/物理專家批判鐵律「沒數字駁回」)。
4. **解法缺席**:有沒有 device/material/method 解法?問題意識要四件套(痛點+解法+材料量化+新穎性),缺則降級。
5. **自證循環**:批判驗證本身會不會「因為查詢含領域詞,結果必含領域詞,所以判有效」的循環論證。

## ★對抗最近性偏誤
- 不得只抽查「剛看到的幾條漂亮 abstract」就放行全體;要對回聲比例做全體統計。

## 輸出
回聲比例 + 領域非瓶頸的 valid 清單 + 量化/解法缺席清單。FAIL 回精準度優先的驗證策略。與 [[validation-false-kill-critique]] 並列批判。
