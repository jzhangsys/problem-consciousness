# 問題拆解:核心問題 → 可定位子問題

## 兩個正交的拆法(都要)
**(A) 尺度層級**(由大到小,定位「在哪一層」):
系統(整體部署/設備/機組)→ 子系統(模組/單元/堆疊)→ 元件(各個界面 / 中間層 / 通道 / 結構件)→ 機制(界面失配 / 傳輸失效 / 極限現象 / 邊界層)→ 物理/理論極限(該機制的理論地板 / 材料塊材極限)。

**(B) 串聯瓶頸鏈**(沿「源頭 → 終點」串聯,定位「哪一段」):
把端到端的過程拆成一連串串聯的阻抗/損耗段(源頭 → 各級界面 → 各級中間層 → 傳輸段 → 終端交換 → 環境)。**每一段串聯阻抗 = 一個獨立子問題**,各有自己的指標與物理上限。

## 子集枚舉
每個骨幹痛點 → 展開成子問題清單(如本專案 problem_clusters 分成 11 群,每群鎖定一個具體瓶頸,如「既有主流方案逼近極限」「動態負載下不穩」「缺乏統一標準」「加料反效果」…)。子問題須繼承父問題的「指標+單位+尺度」(見 `problem-identification`)。

## 紀律
- 跨尺度不可相減(源頭局部尺度的度量 ≠ 系統級尺度的度量)——拆解時標清楚每段尺度。
- 每段子問題標「是串聯主導項還是次要項」(供 `problem-validation` 的 Amdahl 檢查)。
- 輸出子問題樹 → 傳給 `problem-validation` 篩真瓶頸、傳給 `problem-anchored-harvest` 當種子。
