# [A·策略] 框架完整性策略

## 任務
從單一 input 產出**窮舉**的瓶頸清單,不是抽樣。三類齊全:
- **概念問題(機制瓶頸)**:該領域底層的物理/化學機制限制——傳輸阻抗、界面阻抗、擴散/滲流限制、轉換效率上限、非均勻負載、不穩定/臨界現象、驅動代價、各向異性、老化/循環疲勞、瞬態響應等,以領域原生機制表述,**新舊並陳**(既納入老而承重的基礎機制,也納入最新前沿才浮現的機制)。
- **實質問題(器件/材料瓶頸)**:承載上述機制的具體器件、材料、結構、製程、界面——關鍵材料、元件、整合/封裝結構、製造方法等,同樣**新舊並陳**(成熟主力與新興候選並列)。
- **痛點**:把上面綁成「X 機制使 Y 指標超標」的量化敘述。

## 紀律(每條瓶頸都要)
1. **量化指標必備**:寫出該領域原生的量綱(瓶頸值的度量+單位、比率、無因次數,或部署值 vs 理論地板/天花板…)。沒量綱=未成形,退回。
2. **分層標記**:每條標 `layer ∈ {transport基礎傳輸, device器件, reliability可靠度/瞬態}`(供下游 schema)。
3. **provenance 預留**:每條給 `problem_id`,供關鍵字、KG、census 回指。

## 對抗最近性偏誤(★)
- 不准只列你最熟或最近看到的。**逐一掃過外部基準**:領域教科書的瓶頸大項、對應的 CPC/IPC 分類大類、近 3 年領域 roadmap。找資料走 openclaw_workspace(真瀏覽器)。
- 老而承重的(基礎材料/結構瓶頸)與新潮的(最新前沿題目)同等納入。
