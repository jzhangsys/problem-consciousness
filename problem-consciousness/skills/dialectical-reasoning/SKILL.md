# 辯證推理 — 讓對象的內在矛盾逼出框架的下一步

## 定位與立場
在領域研究內、於 [[problem-level-processing]] 的 **L2 概念 / L4 機制 / L5 解法空間** 深化時執行;是 [[meta-problem-layer]]「辯證推理」快篩命中後的續作,不是該元件本身,只把「溶解」裁決回饋給它的 reject/reframe。
**方法環境中立**:規定的否定、否定之否定、揚棄、抽象上升到具體,在唯心(概念自我奠基)與唯物(實踐為尺度)兩讀法下皆成立;**本 skill 預設唯物**——以實踐為尺度、錨定起於對象的真實地基——這是明示立場,非與唯心對稱的口味。拒斥三偽辯證:硬套「正-反-合」三拍、把折衷當「合」、把任意兩物相斥當矛盾。

## 生成內在矛盾(產出程序,不是篩選)
對框架 F:①列 F 成立所必需的界定條件與**明示排除項**;②測「移除哪個排除項會使 F 自身無法界定/運作」——該被依賴的排除項即**候選內在矛盾**(A 在其自身即非-A);③再掘出 F 所**遮蔽**的物質/歷史/利益地基(這是揭露真實 ground,非外插)。
內在性檢(僅約束**反題生成**,不約束合/提升):(a)可回溯到 F 明確排除的具體規定;(b)依賴性——移除它 F 即失效(觀念/形式對象則給概念的內在規定理由)。禁拿無機制關聯的外部反例假冒內在否定。
對立(Gegensatz)兩態:兩極**互相設定、各以對方為成立條件**→就地推展為矛盾續跑;外加而無互相設定→判無生產性,退回 meta。
**主要矛盾**=移除後對象即無法界定/停止發展、或最能解釋當前變化的那一個;餘標次要待轉化。錨定優先序:先物質基礎與真實力量/利益,「機制」只是其領域呈現形式,不得單獨充當物質錨;純形式對象錨於概念內在規定。

## 生成引擎:兩次否定不可混為一談
**規定的否定(第一次否定)**:由 X 的**特定缺陷**確定它過渡到的那一個 Y;輸出**必須指名 Y**,只會說「不對」而給不出下一規定者駁回為抽象否定。
**否定之否定(第二次/絕對否定性)**:對第一次否定的再否定,才回返自身生出保留+提升的具體肯定 W——**非**相加或折衷。三環節=直接規定 →(規定的否定,得 Y)→(否定之否定,回返為更高規定 W)的自我運動。
**抽象上升到具體**:每步引入一個先前被抽象掉的、**可指名的**規定/約束(=「中介」的操作定義);唯物防護:這是在思維中**再生產一個物質上先在的具體總體**,中介須對應對象既有真實規定,非為圓滿而虛構。

## 揚棄 Aufhebung(事後檢查,非先驗要填的格子)
| 判準 | 檢查 |
|---|---|
| 保留 | 正/反題各自站得住的部分是否被收攝(非丟一方)? |
| 否定 | 各自的片面性/絕對化是否被取消? |
| 提升 | 新規定能否**從自身內在必然性自我分化出**正、反題,並說明各自片面性為何是該統一的必要環節?(非外延罩住兩者的「更大袋子」) |
弱化充分形式(免 null-heavy):正/反題成為新規定在**不同條件區間的極限情形**亦算提升。**唯物加驗**:此統一須對應主要矛盾在物質/實踐層的真實(或原則上可實現)解決路徑;純概念涵蓋無實踐可實現性者標 `形式綜合待驗`,不計真揚棄。缺提升=折衷,缺保留=換題,只否定=退回反例。

## 量變到質變
節點原則上可被把握(尺度關係的節線),但**不可用平滑外推/趨勢掃描預測**——新質是機制斷裂非量的延長;改**辨識趨向節點的尺度關係**作預警。質變**指標**:①(首要·客觀)L4 主導機制被另一機制取代(對象真實存在方式的轉化,非同機制的量放大);②(徵兆)L2/L3 指涉物改變(定義失效只是症狀)。互轉雙向:亦問質變如何反過來引起量變。僅在結論做**跨數量級/跨制度邊界**外推時,強制查是否漏質變點。

## 總體性與歷史具體性
最小關係集 {上游依賴, 下游被依賴, 對立項} 只是**起手探針**;須沿**中介鏈**外追,至真值條件回溯到所屬具體構型為止,並指出**孤立哪一條會翻轉真值**。禁把抽象片段當自足結論。
歷史具體性是**方法內環節**:標明此矛盾的**當前歷史規定形態**、及脫離該構型是否即失效(同一「概念」在不同構型中可能已是不同對象)。L1 只存檔佐證,同一性判斷留本 skill。

## 實踐檢驗與改造(唯物收尾)
更高規定不自證,預設以實踐為尺度。ledger **不得合併**兩欄:`否證接觸點`(可經驗推翻,交 L6 執行)與`改造/介入計畫`(praxis 本身:如何介入並改變對象、預期矛盾在改造中如何展開轉化)。[[citation-gate]] 僅於最終主張落地把關引用,勿把實踐等同引用查核。

## 輸出 dialectic_ledger + 微範例 + 硬上限
欄位:`主要矛盾`(錨何真實結構+主要方面)·`內生對立面`(過內在性檢理由)·`合`(勾對三判準)·`規定的否定`(指名 Y)·`質變點`(指標+新質/「無」)·`殘餘矛盾`·`歷史規定形態`·`valid_conditions`(L1 存檔)·`否證接觸點`·`改造計畫`·`conclusion_level`。
**微範例**:F=「以純個體選擇解釋集體結果」→ 移除其排除的「互動的結構效應」則 F 無法解釋協調失靈(依賴性成立)→ 指名 Y=「結構條件下的選擇」→ 否定之否定得 W=「結構化互動選擇」,純個體與純結構各為 W 在不同條件的極限(提升成立)→ conclusion_level=C4。
**硬上限 C4**(機制合理待驗證):未過 L6/實踐檢驗前不得升 C5+,別讓「提升」的氣勢把待驗證命題冒充已確立結論。

## 掛鉤與停止
兩種合法產出:**推進**(生更高規定)或**溶解**(僅在外部反框架內化為規定的否定後**仍生不出更高規定**、且能給**為何夾帶前提**的內在理由時判,回 [[meta-problem-layer]] 取消;純快篩溶解歸 meta)。與 meta「因果懷疑」互查(別把相關對立當機制矛盾);形式/數理對立交 [[critic-formal-math]];需外部證據駁斥交 [[expert-critique-router]] / [[critique-harvest]]。**停止**:一輪只出折衷/抽象否定,或新規定已無可指名的殘餘矛盾且已受實踐檢驗——停,轉 [[problem-framing]] 重框或收束。
