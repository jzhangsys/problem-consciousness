# 問題驗證:真瓶頸 vs 假問題

## 三道閘(全過才是真瓶頸)
**① 物理確定性**(`physics_kernel`,非嘴上痛點)
算這段在**從源到匯的串聯瓶頸鏈**中的**實際佔比**。佔比小 = 改它也沒用。
例:各串聯段用其解析式算貢獻(體阻=厚度/傳導係數、界面阻=界面理論下限、末端交換阻=1/(交換係數×面積))。

**② Amdahl 主導項**
改善這段能讓**最終輸出指標**顯著改善嗎?還是它只是次要項、被別段卡住?
只改次要項 = 有限增益(如某方向判 NEAR_CEILING、某方向判 DEAD_END,皆此關卡擋下)。

**③ 部署落差(headroom)**
- 部署最佳 vs 物理上限的差距大,且研究已逼近上限但未落地 → **真機會**(如某瓶頸:理論地板值 vs 部署值差 4–10×)。
- 部署已逼近上限 → 成熟、增量有限。
- 研究領先但物理上不導向**最終指標**改善 → **死路,剔除**。

## 裁決
REAL_GAP(三閘全過)/ NEAR_CEILING(②過但近上限)/ DEAD_END(②或③不過)。
與 `critic-natural-sciences`、`five-axis-consistency` 互補:本層在**問題端**先篩,避免把假瓶頸帶進下游 harvest/白地。輸出真瓶頸清單 → `problem-anchored-harvest`。
