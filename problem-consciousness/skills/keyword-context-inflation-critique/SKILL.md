# [B·批判] 脈絡膨脹批判

## 立場
假設關鍵字「看起來有效其實太泛」。實證:179 條組合命中 >10 萬件,最寬 26 萬件——這種查詢**不是前沿訊號,是雜訊桶**,後續精準度必崩。

## 攻擊點
1. **泛脈絡詞獵殺**:抓出含過廣脈絡(泛領域名/泛類別名)的 combo,要求換成具體的子領域/應用/結構詞,或砍掉。
2. **寬度即雜訊**:對命中 >10 萬的 combo 一律質疑——它把該瓶頸的精準訊號稀釋掉了。要求收窄(加具體脈絡/結構詞)或降權。
3. **`total>0` 假有效**:批判任何「有命中就算有效」的邏輯——命中數本身不是品質,精準度才是(把球丟給 C 章 [[validation-precision-first-strategy]])。
4. **價值排序顛倒預警**:提醒——精準低命中的前沿 combo(如某具體機制×具體結構的窄查詢,命中 12)價值**高於**泛高命中 combo;生成階段不該因為低命中就少生成它。

## ★對抗注意力稀釋
- 逐條掃泛詞,不抽樣(泛詞 combo 常成批出現,漏一條就放一桶雜訊進 corpus)。

## 輸出
泛詞 combo 清單 + 收窄/刪除建議 + 寬度紅旗。FAIL 回 [[keyword-permutation-coverage-strategy]]。隸屬 [[pc-genesis-loop]] B 章。
