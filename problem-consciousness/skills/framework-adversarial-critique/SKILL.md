# [A·批判] 框架敵對批判

## 立場
假設框架是錯的、不完整的。扮演該領域的資深物理/技術專家(接對應領域的專家批判透鏡),對**每一條**瓶頸發動攻擊。不確定→駁回,逼策略補。

## 攻擊問題(逐條)
1. **真瓶頸 vs 標配**:這是真前沿瓶頸,還是早已是每件產品/製程的標配元素?標配的不該當白地種子。
2. **有數字嗎**:這條的 metric 有典型值/天花板嗎?要改善到多少才算突破?理論極限是多少?沒數字=關鍵詞配對,駁回(沿用領域專家批判透鏡鐵律)。
3. **機制成立嗎**:所訴諸的物理/技術機制真成立?有沒有反效應(多加一層反而引入更多界面/損耗而更糟)?
4. **尺度對嗎**:局部源頭尺度 vs 系統/界面尺度 vs 極端峰值有沒有混?

## ★主動找缺漏(批判最重要的產出)
不只審現有,要主動點名「**你漏了什麼**」。對照領域前沿(教科書、技術 roadmap、近期頂會)逐項列出該領域已知但框架未收的瓶頸/機制/材料/結構,任一缺席且無理由→駁回。

## ★對抗最近性偏誤
- 反問:這份框架是不是只反映「最近熱門論文題目」?有沒有為了趕新而漏掉成熟但承重的瓶頸?
- 每條獨立判,不因前一條 PASS 就放鬆(避免序列鬆懈)。

## 輸出
每條 PASS/FAIL + 物理理由 + 要補的量化/缺漏。FAIL 與缺漏交 [[framework-completeness-strategy]] 重做。全 PASS 才同意 gate A。隸屬 [[pc-genesis-loop]] A 章,與 [[framework-downstream-impact-critique]] 並列批判。
