# [C·監督] 閘對稱性監督

## 職責
查驗證**閘的形狀是否公平**。舊版閘有三個結構性不公,逐一驗:

## 稽核清單
1. **off 不可單筆絕對否決**:舊版 `off==0` 讓「5 筆混 1 筆血泵」殺掉 on=4 的好組合(實證 2 條被誤殺)。要求改成**比例/加權**(off 佔比 ≥40% 才否決),監督檢查是否已改。
2. **on 對低命中放寬**:精準前沿本來命中少、on-title 也少。某精準前沿 combo(命中 12)被 `on>=2` 殺掉。要求**低命中組合降低 on 門檻**(精準度由 abstract 深度閘補),監督檢查。
3. **雙源對稱**:每瓶頸是否兩側都驗?只驗一側即 FAIL(幻影白地風險,見 普查對稱閘(母體對稱/同語域計數))。
4. **★價值排序不可顛倒**:抽查——精準低命中 combo 的最終評分是否 ≥ 泛高命中 combo?若高命中泛桶評分反而高=排序顛倒,FAIL。這是舊版最嚴重的病。
5. **相鄰合法域誤殺**:OFF 濾詞可能與某個合法的相鄰應用域字面重疊(某排除詞誤中一個實為合法的鄰域)。檢查有無誤殺合法相鄰域。

## ★對抗注意力稀釋
- 把「被殺組合」全列出**逐條覆核**(誤殺常藏在尾端),不抽樣。

## 通關條件
off 非絕對否決 ∧ on 對低命中放寬 ∧ 雙源對稱 ∧ 價值排序未顛倒 ∧ 無合法域誤殺。否則回策略。隸屬 [[pc-genesis-loop]] C 章。
