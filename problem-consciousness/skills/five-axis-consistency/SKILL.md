# 五軸自洽骨架
**五軸(每軸都要 harvest 足夠資訊,不可空)**:
1. **痛點/問題**(painpoint_discover + 逐篇拆解 problem)— 是真需求嗎?哪個具體問題?
2. **學術科學**(學術 harvest + academic_layer)— 研究活躍嗎?成熟度?
3. **專利**(CPC + 讀全文 + 跨庫FTO)— 真稀疏還是取樣假象/別名混淆?
4. **市場現有solution + 產業**(harvest-industry-news + density)— 現行解是什麼?趨勢順逆?
5. **法規**(harvest-regulation)— 被封死還是被創造?

**自洽規則**:一個「真機會」需五軸不互相矛盾:
- 痛點真 ∧ 科學活躍 ∧ 專利稀(過FTO)∧ 市場有缺口/順勢 ∧ 法規不封死(或正驅動)。
- 任一軸矛盾 → 標明不洽軸,降級或否決(如某方向:法規軸封死→否決)。
- 法規【創造】的缺口 = 最高價值(如法規新規催生的替代材料/方案)。

**順序**:五軸 harvest+自洽 → 通過者才進拓樸(測地/白地/路徑)。不自洽的不進拓樸。

**終端輸出(每方向一卡,8欄)**:最適合研究?/為什麼/參考什麼/現在是什麼/解決哪個問題/如何解決/誰在研究/哪塊是白地。每欄須有來源。
